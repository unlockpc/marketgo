## 第 0 章　总览：它是怎么工作的

MarketGo 不是几个孤立功能，而是一条**从养号到转化**的流水线，每一步都在防封号：

```
养号(地基) → 健康体检 → 发帖前防封闸门 → 读真帖 → 买家意向打分+闸门
        → 生成"像真人"的回复 → 半自动审核 / 全自动发布 → 线索 → 转化 → 效果分析
```

| 阶段 | 干什么 | 自动化程度 |
|------|--------|-----------|
| 养号 | 模拟真人浏览，把账号养出权重，避免新号一上来就被封 | 全自动（引擎按策略调度） |
| 健康体检 | 查账号是否还登录着 / 是否被限流 / 是否被封 | 全自动（约每 20 小时一次） |
| 防封闸门 | 养号期未满 / 健康异常 / 版块禁推广 / 低意向 → 不发 | 全自动 |
| 关键词回复 | 按关键词搜帖 → 读真帖 → AI 判定意向 + 生成回复 → 带货获客 | 半自动审核 或 全自动 |
| 评论回复 | 回复**自己帖子**下别人的评论（社区运营，低风险） | 半自动 或 全自动 |
| 内容发布 | 自己发帖 / 发文 / 发图文视频，可定时 | 全自动（到点执行） |
| 线索 / 转化 | 每条发出的回复 = 一条线索，跟踪谁回应 / 谁转化 | 记录自动，标记手动 |
| 效果分析 | 漏斗 + 各平台 / 关键词 ROI + SEO 排名 | 自动统计 |

**一个核心引擎驱动一切**：后台有个 Rust "引擎"，每 5 秒循环一次，自动做养号调度、健康体检、定时发布、SEO 采集，并从任务队列里取任务执行。你在界面上做的几乎所有"自动"操作，最终都是往这个队列里塞任务，由引擎按防封规则执行。

## 第 1 章　安装与启动

### 1.1 前置条件

1. **Node.js ≥ 18**。
2. **Rust 工具链**（编译桌面应用需要，见 https://www.rust-lang.org/tools/install ）。
3. **Unzoo 浏览器正在运行**：本工具所有浏览器动作都走 Unzoo（REST API 在 `http://127.0.0.1:9399`）。先确保 Unzoo 已启动，否则养号 / 回复 / 发布全部跑不动。

### 1.2 启动桌面应用（推荐）

在项目根目录执行：

```bash
npm install
npm run tauri:dev      # 开发模式运行（带界面）
```

`tauri:dev` 会自动起前端静态服务器（端口 1420，即 `npx serve dist/tauri -l 1420 -s`），再打开窗口。

打包安装版：

```bash
npm run tauri:build    # 生成安装包（当前打包目标为 Windows .msi / NSIS .exe）
```

### 1.3 白屏 / "localhost 拒绝连接" 怎么办

如果窗口里写 "localhost 拒绝连接 / ERR_CONNECTION_REFUSED"：说明是用 `cargo run` 起的、但 1420 服务器没起。两种解法：

- 改用 `npm run tauri:dev`；或
- 单独起一个静态服务器再点窗口里「刷新」：
  ```bash
  npx serve dist/tauri -l 1420 -s
  ```

> 引擎是 Rust 写的、独立运行；白屏只是网页没加载到，不影响后台引擎。

### 1.4 让引擎开机自动跑

设置环境变量 `UNMARKET_ENGINE_AUTOSTART=1` 再启动应用，引擎会自动开始循环；否则需要在控制台点「🟢 一键全自动」或在任务页点「▶ 启动引擎」。

### 1.5 数据存哪

数据库在系统应用数据目录下 `unmarket/unmarket.db`：

- Windows：`C:\Users\<你>\AppData\Roaming\unmarket\unmarket.db`
- macOS：`~/Library/Application Support/unmarket/unmarket.db`
- Linux：`~/.local/share/unmarket/unmarket.db`

（账号 / 身份 / 产品 / 任务 / 线索 / 配置 全在这里。）

## 第 2 章　首次配置（最短上手路径）

第一次用，按这 4 步走最省事，10 分钟内能跑起来：

1. **配 AI Key**（生成回复、意向打分要用）
   → 进 **⚙️ 设置** → 「🤖 AI Configuration」→ 选 provider（默认 **Google Gemini**，有免费额度）→ 填对应 Key → 「🔄 刷新模型列表」选模型 → 可点「Test Connection」验证 Key+模型是否可用 → 「Save AI Settings」。

2. **建邮箱身份并登 Gmail**（多号隔离的地基）
   → 进 **📧 邮箱账号** →（可选）先点机场代理条「设置机场订阅」给每个邮箱分配独立 IP → 点「+ 新建 Gmail」填一个真实 Gmail（自动建浏览器 + 随机指纹 + 分配出口节点，并自动开 Google 登录引导）→ 在该邮箱面板点「📧 登录Gmail」，在弹出的浏览器里登好这个 Gmail。

3. **逐平台开通账号**
   → 还在该邮箱面板上，点「🚀 检查并开通账号」：逐平台检查——有就登录、没有就注册（含自动选账号授权），账号自动挂到这个 Gmail 名下，共用它的浏览器 / IP / 指纹。

4. **建产品**
   → 进 **📦 产品管理** → 「📦 + Add Product」→ 填产品名 + URL + 一句话标语（回复里软提产品时用）→ 「Save Product」。

做完这 4 步，就可以去 **🎛️ 控制台**（Dashboard）点「🟢 一键全自动」开跑了（强烈建议第一次先开**演练模式**，见 [第 4 章](#第-4-章️-控制台dashboard)）。

> 不配 AI Key、不登平台也能启动，但：未登录 → 养号 / 回复会被判"未登录"拦下；没 AI Key → 回复退化成模板（不推荐）。

## 第 4 章　🎛️ 控制台（Dashboard）

控制台是日常"一屏看全 + 一键开关"的总览页。

### 4.1 一键开关

- 右上角主按钮「🟢 一键全自动」：点一下切换引擎启 / 停。引擎在跑时这里会反映状态。
- 这是最省事的开关；和任务页的「▶ 启动引擎 / ⏸ 停止引擎」是同一个引擎。

### 4.2 自动驾驶状态（Autopilot）

显示引擎当前是否在跑、已处理多少任务等实时状态。

### 4.3 待办提醒（Attention）

把需要你出手的事汇总成卡片，每条带一个「去…」按钮直接跳到对应页，例如：

- ⚠️ 账号掉登录 / 被封 → 去账号页重登
- 🧩 有账号还没配 profile → 去配
- 📝 有回复在等审核 → 去审核队列
- ⛔ 有任务被阻塞 → 去看原因

### 4.4 🛡️ 安全设置（设一次，引擎永久遵守）

点开「🛡️ 安全设置 — 设一次即可，引擎自动遵守，不用每次确认」折叠区，有 5 项：

| 控件 | 选项 / 范围 | 说明 |
|------|------------|------|
| **回复模式** | 全自动（直接发，无人值守） / 半自动（生成后我审核再发） | **默认半自动**。新手务必先半自动。 |
| **养号期保护** | 勾选框 | **默认开**。新号先只养号，养熟自动解锁对外。"关掉=新号立刻对外，风险高"。 |
| **演练模式** | 勾选框 | 开了之后只跑流程、打日志、**不真实发布**。第一次跑强烈建议先开。 |
| **意向阈值** | 0–100 数字 | 低于此分不回复，**默认 40**。调高=更挑、更省 AI 额度。 |
| **安静时段** | 起 / 止两个整点（0–23） | 该时段不**对外发**（养号 / 体检照常）。留空=不限。跨夜可设 22→6。 |

> 这几项就是引擎全局行为开关，改完即时生效，引擎自动遵守，不用每次确认。它们也能在任务页改（同一套配置）。

### 4.5 📈 今日自动战果

展示当天引擎的 KPI（发了多少、回了多少、养了多少号等）。

### 4.6 ⚙️ 更多 / 高级页

折叠区里是几个快捷跳转按钮：任务 / 审核 / 养号 / 分析 / 线索、市场提交、推广活动 / 关键词、产品、账号 / 绑定 profile、设置 / AI Key / 全局 profile。

## 第 5 章　📧 邮箱账号（上）邮箱视角 · 身份 · 机场

> **这是多号隔离的地基，强烈建议从这里开始建号。** 旧版独立的「🧑‍🤝‍🧑 身份隔离」页已并入本页——建 Gmail 身份、登 Gmail、检查并开通账号、机场代理，现在全在「📧 邮箱账号」一页搞定。

核心理念：**一个 Gmail = 一套独立身份** = 独立浏览器（cookie / 登录态）+ 独立出口 IP（机场节点）+ 独立指纹。同平台要多个号？就多建几个 Gmail——它们互不关联，平台看不出是同一个人，防关联封号。

### 5.1 页面布局：邮箱视角

整页是"**邮箱视角**"——选哪个邮箱、下面就只显示那个邮箱名下的账号。从上到下三块：

1. **🌐 机场代理条**（最上）：见 5.2。
2. **📧 邮箱切换条**：一行 tab，每个真实 Gmail 一个「📧 邮箱地址」tab；还没归属到任何邮箱的账号汇总成「🧩 未归属(N)」tab。点 tab 切换；最右是「+ 新建 Gmail」。
3. **当前邮箱面板**：选中邮箱的信息条（邮箱地址 / 节点地区 / 独立浏览器 profile / 账号数）+ 操作按钮（见 5.4）+ 该邮箱名下的账号卡片列表。

### 5.2 🌐 机场代理（出口 IP 来源）—— 可选但推荐

- 页面最上方「🌐 机场代理」条显示节点池状态：已配置时显示"节点池 N 个（空闲 M）· 每个邮箱分一个独立出口 IP"；未配置显示"未配置——配了才能给邮箱分配独立 IP"。
- 点右侧「设置机场订阅」/「换/刷新订阅」，在弹框粘贴你的**机场订阅链接（Clash 订阅）**即可拉取节点。
- 工具自带独立内核（Mihomo），与你日常用的 Clash Verge 互不干扰；节点不直接暴露、仅本机使用。
- 配好后，后续每新建一个 Gmail 身份会自动从节点池分一个独立出口 IP 给它。不配机场也能用（走本机 IP），但多号共用一个 IP 风险更高。

### 5.3 新建 Gmail 身份

1. 点邮箱切换条右侧「+ 新建 Gmail」。
2. 在弹框输入一个**真实 Gmail**（如 `yourname@gmail.com`）。
3. 约 5–10 秒，自动完成：建独立浏览器 profile（命名取邮箱本地部分，如 `Profile_lixd220`）→ 随机指纹 → 分配一个出口节点 → 绑定，**并自动打开 Google 登录引导**让你登这个 Gmail。建好后自动切到这个新邮箱 tab。
4. （若同名 profile 冲突，会自动换唯一后缀重试，不用管。）

### 5.4 当前邮箱面板的操作按钮

选中某个真实邮箱后，信息条下方有一排按钮：

| 按钮 | 作用 | 顺序建议 |
|------|------|---------|
| **📧 登录Gmail** | 打开这个邮箱的浏览器去登它的 Gmail（基础登录） | **第 1 步**，先登 Gmail |
| **🚀 检查并开通账号** | 逐平台检查：有就登录、没有就注册（含自动选账号 + 授权），账号自动挂到这个邮箱名下 | **第 2 步** |
| **+ 加账号** | 手动给这个邮箱加一个平台账号（打开"添加账号"弹窗，自动归属本邮箱） | 按需 |
| **删除此邮箱**（右侧红字） | 删除这个邮箱身份：释放节点、删 profile、解绑名下账号 | 谨慎 |

面板下方列出该邮箱名下所有账号卡片（操作见 [第 6 章](#第-6-章-邮箱账号下账号卡片--养号--profile)）。

> **新建邮箱后的标准动作**：先「📧 登录Gmail」→ 再「🚀 检查并开通账号」。两步走完，这个 Gmail 名下各平台账号就都登好了、共用同一套浏览器 / IP / 指纹。

## 第 6 章　📧 邮箱账号（下）账号卡片 · 养号 · profile

本章接第 5 章，讲「📧 邮箱账号」页里**每张账号卡片**的操作、养号弹窗、手动加账号、以及 profile 解析规则。

> 说明：旧版页面顶部的"账号健康总览大圆环"和"全局自动注册区（选平台 + Auto-Login/Register + Sync All）"在新版**已隐藏/取消**——健康状态改为直接显示在每张账号卡上，注册改走邮箱面板的「🚀 检查并开通账号」。

### 6.1 每张账号卡的操作

每张卡显示：平台图标 + 名、用户名、邮箱、状态徽标、健康指示、所属邮箱身份。按钮（按账号状态出现不同的）：

| 按钮 | 作用 |
|------|------|
| **🔑 Auto-Login** | 自动登录：查登录 → Google 登录 → 否则注册 |
| **🌱 Quick Nurture** | 打开养号弹窗，给这个账号手动养一次（见 6.4） |
| **🔥 Start Warmup** | 仅新号显示，立刻入队一次养号任务 |
| **Create Profile** | 该号还没绑 profile 时显示，给它新建一个 Unzoo 浏览器 profile |
| **🛡️ / ⚡ Toggle Stealth** | 有 profile 时显示，开 / 关该 profile 的隐身（反检测）模式 |
| **🎭 Randomize Fingerprint** | 有 profile 时显示，随机化该 profile 的浏览器指纹 |
| **🌐 Set Proxy** | 有 profile 时显示，给该 profile 单独设代理（打开代理弹窗） |
| **Delete** | 删除该账号 |

### 6.2 账号养护弹窗（🌱）

点「🌱 Quick Nurture」打开「Account Nurturing / 账号养护」弹窗：

1. 选**浏览时长**：30 秒 / 1 分钟（默认）/ 2 / 5 / 10 分钟。
2. 点「Start Nurturing / 开始养号」：进入进度阶段——大号倒计时 + 进度条 + "浏览中…"。
3. 中途可「Stop / 停止」。
4. 完成显示 ✅ "养号完成"，点「Close / 关闭」。

> 养号 = 模拟正常用户浏览（滚动、停留、点赞、关注），提升账号权重、降低被封风险。日常你**不用手动养**——引擎会自动按策略调度（见 [第 10 章 养号调度](#108-养号调度)）；这里的手动养号用于"想立刻催一把"。

### 6.3 手动添加账号弹窗

在当前邮箱面板点「+ 加账号」（页头那个旧的「+ Add Account」已隐藏）：选平台（Twitter/X、LinkedIn、Reddit、Facebook、Dev.to、Medium、Discord、Telegram、Mastodon、Weibo）→ 填用户名 / 邮箱 → 填密码 / API Key → 「Save Account」。账号会自动归到当前选中的那个邮箱名下。

> 工具**不替你保管平台密码做明文登录**——它主要用的是"已登录浏览器 profile 的登录态"。手动加账号主要用于 API 直发类平台（Discord webhook / Telegram bot 等）或登记信息。

### 6.4 账号的 profile 是怎么解析的（重要）

执行任务时，引擎按这个优先级给账号找浏览器 profile：

```
账号所属身份(persona)的 profile
  > 账号自身的旧式直接绑定
  > 该平台任意一个已绑 profile 的账号
  > 全局默认 profile（设置页选定）
```

四者都没有 → 报"请给账号绑定 profile，或设一个全局默认 profile"。所以：**要么把账号挂到身份下（推荐），要么在设置页连一个全局默认 profile 兜底。**

## 第 10 章　📋 任务队列（Tasks）— 引擎主控台

**这是日常运营的核心页**：引擎启停、回复模式、审核队列、养号调度、效果分析、线索、选择器自检全在这。

### 10.1 引擎开关与状态

- 页头显示「引擎: <状态>」。
- 「▶ 启动引擎」/「⏸ 停止引擎」：启停后台引擎循环。
- 「🌱 Create Nurture Task / 新建养号任务」：打开批量养号弹窗（见 10.9）。
- 「Clear Completed」：清掉已完成任务。
- 任务统计：Pending（等待）/ Running（执行中）/ Completed（完成）三个计数。

### 10.2 💬 回复模式 + 审核队列

- 下拉「💬 回复模式」：
  - **半自动 — AI 生成后人工审核再发**（review，默认）
  - **全自动 — AI 直接发布（无人值守）**（auto）
- 说明语："引擎读取真实帖子 → AI 判定相关性 → 生成针对性回复"。
- 「待审 N」徽标 + **审核队列**（`reviewList`）：列出所有待批准回复，**按意向分从高到低排序**，每条带原帖、AI 生成的回复文案、AI 理由、意向分。你可以：
  - **直接编辑文案**
  - **批准发布**（approve）→ 引擎去发
  - **驳回**（reject）→ 标记拒绝、不发

> 半自动模式下，引擎生成的对外回复**一律进这个队列等你点批准**，不会自己发。这是新手最该用的模式。

### 10.3 任务列表与任务状态

任务列表列出全部任务（类型 / 状态 / 进度 / 结果 / 时间），可对单个任务操作。任务状态含义：

| 状态 | 含义 | 你能做什么 |
|------|------|-----------|
| ⏳ pending | 等待执行 | 等引擎取 |
| ⚡ running | 执行中 | — |
| ✅ completed | 完成 | — |
| ❌ failed | 失败（重试超限） | 重试 / 看日志 |
| ⚠️ blocked | 被防封闸门拦下（看原因） | 解除阻塞 / 调策略 |
| 🚫 cancelled | 已取消 | 重试 |

对阻塞 / 失败 / 取消的任务可「解除阻塞 / 重试」，对等待 / 阻塞的可「取消」。

### 10.4 引擎都跑哪些任务

引擎从队列取的任务类型：
- **nurture（养号）**：模拟真人浏览，提权重。
- **health_check（健康体检）**：登录态 / 限流 / 封禁检测，更新健康状态。
- **reply_keyword（关键词回复）**：搜陌生人帖 → 判相关 / 意向 → 生成回复 → 审核或自动发。
- **reply_mention（评论回复）**：回**自己帖子**下的评论（目前仅 Reddit 能抓评论，其余平台优雅阻塞）。
- **post / content_publish（内容发布）**：发原创文 / 图文 / 视频。
- **metrics_collect（成效采集）**：SEO 排名 + 品牌提及。

### 10.5 引擎循环逻辑（了解即可）

引擎每约 5 秒一轮：释放卡死的 running 任务 → 养号调度 tick → 健康体检 tick → 定时发布 tick → 成效采集 tick → 判断是否安静时段 → 原子地取一条 pending 任务 → 执行 → 落结果（成功 / 阻塞 / 重试 / 失败）→ 任务间留约 2 秒间隔。

### 10.6 📊 效果分析

「📊 效果分析」卡：发现 / 跳过 / 回复 / 线索 / 转化漏斗，按平台 + 关键词看哪儿 ROI 高。

### 10.7 🎯 线索 / 转化（轻 CRM）

「🎯 线索 / 转化」卡："每条发出的回复 = 一条线索，跟踪谁回应了、谁转化了"。线索列表每条显示原帖摘录、发出的回复、对方回应（若有）、转化状态、时间。你可手动标记**转化 / 已回应 / 忽略**。

> 注意：对方"是否回应"目前**靠手动标记**，自动回访检测尚未做。

### 10.8 🌱 养号调度

「🌱 养号调度」卡："引擎按平台策略 + 号龄分期，活跃时段自动安排浏览养号；每日体检账号健康度。"内容区按账号显示：分期 / 号龄 / 今日 x/目标 / 累计时长 / 健康徽标。

**号龄分期与每日配额**（每个账号按注册至今的天数自动分期）：
- **养号期（新号，0 ~ warmup_days 天）**：每天场次 = 策略下限（最保守）。期内**禁止对外带货回复 / 发帖**，只养号 + 回自己帖评论。
- **成长期（warmup_days ~ 2×warmup_days）**：每天场次 = （下限 + 上限）/ 2。
- **成熟期（≥ 2×warmup_days）**：每天场次 = 上限。

调度还会遵守：账号健康（封 / 掉登录 / 限流 的账号跳过）、活跃时段、最小间隔（活跃窗口 / 目标场次，至少 20 分钟）、今日是否已达标。

### 10.9 批量养号弹窗

点「🌱 新建养号任务」：勾选要养的账号（可全选 / 取消）→ 选每个账号时长（30 秒 ~ 10 分钟）→ 可勾「持续运行（养完一轮间隔 5 分钟继续下一轮）」→ 「🌱 开始任务」。

### 10.10 🔍 选择器自检

平台改版后回复 / 发布会"找不到帖子 / 找不到按钮"，用这个排查：
- 填**平台**（reddit / twitter / linkedin / producthunt / hackernews…，默认 reddit）+ **关键词**（默认 AI tools）。
- 点「检测」→ 结果区（等宽字体）显示实时命中数 + 样本，校准帖子选择器。
- 命中 0 多半是**没登录**（撞登录墙）或**平台改版**——先登录再测；改版就要更新该平台选择器。

3. **市场列表**：列出各市场及其配置状态。点某个市场的「View/Generate Listing」→ 弹「上架资料」窗，显示按该市场格式生成的资料（描述 / 特性 / 安装说明等）。
4. 在弹窗里「Copy」复制资料去手动提交，或（支持的市场）自动提交 / 预填表单。
5. 提交状态会流转：materials_ready（资料就绪）→ submitted（已提交）→ listed（已上架）。

## 第 14 章　📈 成效追踪（SEO 排名 + 品牌提及）

每天自动用专用的 **auto / Default** profile（与发帖账号隔离，排名最干净）走 Unzoo 采集三类指标。地区固定 `gl=us / hl=zh-CN / pws=0` 保证历史可比。

### 14.1 三类追踪关键词

- **品牌词（守，brand）**：你的品牌词排名，应永远 #1。
- **长尾词（攻，longtail）**：目标长尾词排名，越低（越靠前）越好。
- **品牌提及（领先，mention）**：第三方独立域名提及你的数量，是领先指标。

### 14.2 操作

1. 顶部「刷新」/「🔍 立即采集」（手动触发一次采集任务）。
2. **采集设置卡**：可勾「启用 Google Trends 采集」（小品牌通常无数据，默认关）。
3. **追踪关键词卡**：填关键词（如"Claude Code 做产品"）→ 选类型（长尾 / 品牌 / 品牌提及）→ 填目标域名 →「+ 添加」。列表显示每个词的排名位次、变化、上次检查时间，可编辑 / 删除。
4. **指标总览**：品牌词排名、长尾词排名、品牌提及数、（开了 Trends 的话）趋势，以及随时间的排名历史。

> 采集默认每天一次（由引擎按 `metrics_interval_secs` 节流，默认 86400 秒），引擎在跑时自动进行。

## 第 16 章　⚙️ 设置（Settings）

### 16.1 🤖 AI Configuration

- **Default AI Provider**：Google Gemini（⭐推荐，有免费额度）/ OpenAI / DeepSeek / 阿里千问。
- Key 输入框**只显示当前下拉选中的那个供应商**（不再 4 个平铺）：选 Gemini 就只显示 Gemini Key 框（`AIza…`），切到 OpenAI / DeepSeek / 千问就换成对应的（`sk-…`），每个都附"去哪拿 Key"的链接。各 provider 的 Key 仍**单独存**，切换不会丢。

- 显示 Unzoo 连接状态。
- **Browser Profile**：选一个 profile → 「Connect」连接，它会成为**全局默认 profile**（供没有任何归属的账号兜底）。「🔄」刷新 profile 列表。
- **Unzoo Path**：显示检测到的 Unzoo 安装路径（找不到显示"未找到"）。

### 16.3 Language / 语言

界面语言：简体中文 / English。

### 16.4 Scheduler（调度器）

- Mode：Round Robin / Weighted / Priority / Smart(AI)。
- Interval（分钟，默认 60，最小 5）、Max Daily Posts（默认 50，最小 1）。
- 「Save Scheduler Settings」保存。

### 16.5 🌐 Proxy Pool（代理池）

- 统计：Total / Active / In Use / Failed。

## 第 17 章　防封机制：为什么它有时"不发"

工具默认**保守**，宁可不发也不冒险被封。被拦时任务变 `blocked` 并写明原因：

1. **养号期未满**："养号期未满（x/y 天）" → 等号养熟，或在养号策略调小该平台 `warmup_days`，或临时关「养号期保护」总开关。
2. **健康异常**："账号健康异常（logged_out / shadowbanned / banned）" → 去该账号所属身份的浏览器重新登录。
3. **版块禁推广（Reddit）**：发评论前查 `/r/<版块>/about/rules.json`，禁自我推广 / 广告的版块直接跳过。
4. **低意向**：意向分 < 阈值（默认 40）→ 跳过，省 AI 额度。
5. **安静时段 / 回复间隔**：安静时段内不对外发；回复间最小间隔也会校验。
6. **熔断保护**：某平台连续报错（被封 / 限流 / 403 / 429）会触发熔断，按严重度冷却（致命 24 小时、严重 2 小时、警告 30 分等），冷却期内跳过该平台任务。

**健康状态 health_status**：`healthy`（健康）/ `logged_out`（掉登录）/ `shadowbanned`（限流）/ `banned`（封禁）/ `unknown`（待配置）。掉登录 / 限流 / 封禁的账号会被自动跳过。引擎约每 20 小时给每个账号体检一次。

> 这正是相对纯 API 类工具的优势：真实浏览器 + 养号 + 多重闸门，更难被风控。

## 第 19 章　常见问题 FAQ / 排障

- **Q：界面白屏 / localhost 拒绝连接？**
  A：用 `npm run tauri:dev`，或单独 `npx serve dist/tauri -l 1420 -s` 后点刷新（见 1.3）。

- **Q：回复任务全被 blocked "养号期未满"？**
  A：符合预期——新号先养。等号龄到 warmup_days，或临时把该平台 warmup_days 调小测试，或在控制台关「养号期保护」。

- **Q：提示"未登录 xx 平台"？**
  A：去「📧 邮箱账号」页，切到该账号所属的邮箱 tab，点「📧 登录Gmail」登好 Gmail（或在账号卡片点「🔑 Auto-Login」），再让健康体检 / 养号重跑。

- **Q：选择器自检命中 0？**
  A：多半是没登录（撞登录墙）或平台改版。先登录再检测；改版就更新该平台的帖子选择器。

- **Q：回复内容质量怎么看？**
  A：开演练模式，日志会打印 AI 真实生成的回复 + 意向分 + 理由（`[ENGINE] [平台] 干运行: …`）；或直接看任务页审核队列。

- **Q：怎么彻底不自动发？**
  A：保持半自动（review，默认）——所有对外回复都要你在审核队列点「批准发布」。

- **Q：定时发的内容到点没发？**
  A：定时发布依赖引擎在跑——确认引擎已启动（控制台「🟢 一键全自动」或任务页「▶ 启动引擎」）。

- **Q：报"无可用 profile"？**
  A：把账号挂到一个身份下（推荐），或在设置页选一个 profile 点「Connect」设为全局默认 profile 兜底。

## 第 20 章　当前能力边界（实话实说）

**已实现并验证：**
- 身份隔离：一个 Gmail = 独立浏览器 + 独立出口 IP（机场节点）+ 独立指纹；名下各平台账号共用。
- 账号自动登录 / 注册（查登录 → Google 登录 → 否则注册）、逐平台"检查并开通"。
- 养号调度引擎（号龄分期 + 活跃时段 + 间隔均摊）、账号健康体检（轮询、修了 SPA 误判）。
- 发帖前防封闸门（养号期 / 健康 / Reddit 版规 / 意向 / 安静时段 / 熔断）。
- 关键词回复：搜帖 → 读真帖 → 意向打分 → 生成"针对本帖、像真人、按需软提产品"的回复（Reddit / Twitter 质量良好，能按帖子语言作答）。
- 半自动审核队列（带意向分、可编辑、批准 / 驳回）、全自动发布。
- 内容发布（文本 + 中文图文 / 视频）+ 定时发布。
- 线索 / 转化轻 CRM、效果分析漏斗、统计看板。
- 成效追踪（SEO 排名 + 品牌提及，专用隔离 profile）。
- 市场提交（MCP / Skill 上架资料生成）。
- 选择器自检；Reddit / Twitter 选择器已校准；LinkedIn 走"信息流就地评论"。

**边界 / 待接续：**
- 平台覆盖逐步增加；选择器需登录态，平台改版需重新校准。
- `reply_mention`（回自己帖评论）目前**仅 Reddit** 支持抓评论，其余平台优雅阻塞。
- LinkedIn 浏览器自动化风险较高（平台 2025 封锁过此类），建议低频 + 充分养号；其信息流帖无固定 URL，用"定位串"去重。
- 线索的"对方是否回应"目前**手动标记**，自动回访检测待做。
- 代理池（`proxies` 表）可增删测，但**尚未接入对外发布引擎**；多号独立 IP 走身份隔离 + 机场节点。
- 全自动模式真实放量前，建议先用**你自己可控的帖子**点一次「批准发布」，确认发布动作 OK 再放量。
